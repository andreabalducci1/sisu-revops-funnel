# Copy — Copywriting du tunnel

Écrire un copywriting qui convertit pour chaque page.

## Prérequis
Lire : `memory/identity/business.md`, `memory/identity/offer.md`, `memory/funnel/strategy.md`, `memory/knowledge/frameworks.md`.

## Workflow

1. Demander sur quelle page travailler (ou "tout").
2. Pour chaque page, appliquer les frameworks (`frameworks.md` — AIDA / PAS / BAB + anatomie de page) :
   - **Landing** : eyebrow, headline (bénéfice, pas feature), subhead, 3 bullets de valeur, CTA, preuve sociale.
   - **Ressource** : eyebrow, titre, description (délivrer + créer le désir de l'appel), CTA vers réservation.
   - **Réserver** : eyebrow, headline, description (rappeler le bénéfice de l'appel + rassurer).
   - **Merci** : eyebrow, headline, body, 3 prochaines étapes.
3. Proposer 2-3 variantes pour le headline de la landing (l'élément le plus important).
4. Valider avec l'utilisateur.
5. Sauvegarder dans `memory/funnel/copy.md` ET appliquer dans `config.ts`.

## Règles
- Français, ton adapté à la cible (lire `business.md`).
- Un seul CTA par page.
- Pas de spam words, pas de fausses promesses.
- Concret et spécifique > générique.
- Après validation, proposer `/design` (si pas fait) puis `/funnel`/`/preview`.
